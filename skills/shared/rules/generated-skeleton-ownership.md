# Generated Skeleton Ownership

## Always True

- Treat cap4k skeletons as generated by cap4k.
- Put complex business logic inside generated skeleton surfaces.
- Require an explicit technical design exception before creating parallel handwritten structure.
- Preserve generated-versus-handwritten ownership in every path decision.
- Keep runtime guidance self-contained.

## Drift Checks

- Prevent "missing skeletons should be handwritten to make progress."
- Prevent "parallel structure is fine if names are similar."
- Prevent "src/main means handwritten ownership."
- Prevent "business logic belongs next to generated code instead of inside approved surfaces."
